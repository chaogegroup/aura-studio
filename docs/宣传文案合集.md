## 一、掘金 / 少数派（技术社区图文）

> 调性：有技术深度但不晦涩，面向「懂一点技术的创作者」，突出开源 + 实用 + 架构亮点。
> 标题可选：
> - 「开源了一个 AI 多模态创作工作站：文本对话 / 文生图 / 文生视频 / 无限画布，MIT 协议」
> - 「不想在 10 个 AI 工具间来回切？我做了个全合一的桌面端创作台，开源了」

### 正文

最近 AI 工具越来越多——聊天用一个、画图用一个、做视频又用一个，素材散落各处，来回切换很割裂。于是我做了一个桌面端的 AI 创作工作站 **AURA Studio**，把这些能力整合到一个界面里，并且**完全开源**（MIT 协议）。

【截图位：主界面全貌】

### 它能做什么

一句话：**文本对话、图像生成、视频生成、无限画布、可进化的 AI 助理**，五合一。

**1. 文本对话** —— 流式响应、思考过程可视化（灰色斜体展示推理链）、深度思考开关、多轮会话管理。支持多模型切换，右边栏模型卡片点一下就换。

【截图位：文本对话 + 思考过程】

**2. 图像生成** —— 文生图（T2I）和图生图（I2I）都有。分辨率不用填数字，点可视化预设按钮（1152×768、3:2 这种）；中文提示词自动翻译成英文再提交；还有模板预设一键填充。

【截图位：文生图结果】

**3. 视频生成** —— 文生视频，提示词可选自动翻译，任务提交后在任务面板轮询，完成自动展示结果和下载链接。

【截图位：视频生成任务】

**4. 无限画布** —— 这是个人最喜欢的功能。生成的图、文、视频都可以丢到画布上自由排布，拖拽、缩放、组合灵感。比起在文件夹里翻素材，画布更像创作者真实的思考方式。

【截图位：无限画布】

**5. AI 助理（Agent）** —— 不只是聊天，是个能多步推理、调用工具的 Agent。它有向量记忆（记得之前聊过什么）、知识库 Wiki、技能系统（SKILL.md 格式，可让它学会新技能），甚至能自我进化。让它「生成一张赛博朋克城市的图，再把结果整理成文案」，它能自己拆步骤完成。

【截图位：Agent 多步推理】

### 技术栈（给开发者看的）

如果你关心实现，架构很轻量，**没有前端框架、没有打包转译，源码即所见**：

| 层 | 技术 |
|---|---|
| 前端 | 原生 HTML + CSS + Vanilla JS |
| 后端 | Python FastAPI |
| 桌面壳 | Electron + PyInstaller |
| AI API | Agnes AI |
| 存储 | 又拍云 S3 兼容 + localStorage |

Agent 核心能力基于开源项目 CowAgent（MIT, Copyright zhayujie）移植适配，已在 `THIRD_PARTY_NOTICES.md` 声明。整个项目依赖都是 MIT 兼容，欢迎贡献。

### 怎么用

两种方式：

**直接下安装包**（Windows）：到 GitHub Release 下载 `AURA-Studio-2.4.0-Setup.exe`，装完打开，在设置里填自己的 API Key 即可。

【GitHub链接】

**源码运行**（任意平台）：
```bash
git clone https://github.com/chaogegroup/aura-studio.git
cd aura-studio
pip install -r requirements.txt
cp .env.example .env   # 填入 Agnes API Key、又拍云密钥
AURA_DEV=1 python backend/main.py   # 浏览器访问 http://127.0.0.1:18922
```

> 🔒 所有密钥都由你自己填写，仓库不含任何凭据。API Key 用 Agnes AI，图床用又拍云（也可换其他 S3 兼容存储）。

### 为什么开源

AI 工具不该被锁在收费墙后面。把它开源，一是希望更多人能用上顺手的创作工具，二是希望社区一起把它做得更好——加新模型、新画布节点、新 Agent 技能。MIT 协议，随便用，欢迎 PR。

如果你觉得有用，给个 ⭐ 就是对我最大的鼓励。有问题或建议，直接开 Issue。

## 五、发布节奏建议

1. **先发技术社区**（掘金/少数派）：长图文建立信任，沉淀搜索流量。
2. **同步发即刻/朋友圈**：短文案引流，让身边人先知道。
3. **次日发小红书**：用前一天的截图，吃长尾搜索流量。
4. **视频单独排期**：B站/抖音需要录屏剪辑，可晚 1-2 天发，但传播力最强。
5. **公众号压轴**：综合长文，适合周末发，配合社群转化。

每篇都带上 GitHub 链接，并在仓库 README 顶部加上各平台文章链接（互相导流）。
